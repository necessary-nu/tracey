# Tracey User Guide

Tracey maintains traceability between specifications and code. You write requirements in markdown, annotate your source code with lightweight comments, and tracey tells you what's covered, what's missing, and what's gone stale.

## Guide Contents

1. **[Getting Started](getting-started.md)** — Install tracey and get your first coverage report in minutes.
2. **[Writing Specs](writing-specs.md)** — Define requirements in markdown specification documents.
3. **[Annotating Code](annotating-code.md)** — Reference requirements from source code comments.
4. **[Configuration](configuration.md)** — Set up `.config/tracey/config.styx` for your project.
5. **[Dashboard](dashboard.md)** — Browse specs, coverage, and sources in the web UI.
6. **[Editor Integration](editor-integration.md)** — Real-time feedback via LSP in Zed, VS Code, and other editors.
7. **[AI Integration](ai-integration.md)** — Use tracey with Claude Code and other MCP-capable assistants.
8. **[Versioning](versioning.md)** — Track requirement changes and keep code in sync.
9. **[CLI Reference](cli-reference.md)** — Every command, flag, and subcommand.
